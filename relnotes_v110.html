Whats New

New Management features

vMIST (vitalpointz Manged IoT) Service

Introducing a new, centralized service that provides below functions to every IoT Core / Core-lite droplet Launched.

- When a IoT Core/Core-lite is installed on a new droplet, an access URL is created & registered with Public DNS.

- A CA signed Letsencrypt certificate is generated and installed into the droplet.

- Certificate issued by Letsencrypt is periodically checked for validity, if the expiry is imminent, certificate gets renewed automatically.

- Provides easy backup/restore tools and storage spaces to store and recover the IoT Core/Core-lite data in the cloud.

- Upon request from customer, a SSH key pair can be installed to customer droplet enabling a support-engineer to have temporary remote access to the droplet for a debug/support session

- Remote Monitoring service, which provides Vitalpointz managed-service-team a dashboard of performance statistics for every customer droplets.

- A Software update service for IoT Core/Core Lite instances

Note: Subscription to vMIST Service is mandatory and free to customers. vMIST service runs as part of vitalpointz service offering and does not require additional licensing or purchase of resource from Digitalocean.

Back up & Restore

This feature is available in the CLI of the droplet. Login to CLI using SSH, type backup. This will archive the essential data from the running IoT Core/Core Lite, encrypts them, saves under vitalpointz Managert IoT Service (vMIST) Server.

Remote Debug

This feature is available in the CLI of the droplet. Login to CLI using SSH, type remote-debug enable. This will enable remote SSH access of the droplet to vitalpointz support team. The SSH Key is autogenerated from vitalpointz Managet IoT Service (vMIST) server, public key stored on the droplet, private key will be made available to vitalpointz support staff securely. Once enabled, the SSH key will be active for 72hours. Post that, the key is automatically removed from the droplet.

If the user wants to terminate the remote debug session before 72 hour duration, they can run remote-debug disable to stop the same.

At any time, type remote-debug status to see the status.

Remote Monitoring

This feature is available in the CLI of the droplet. Login to CLI using SSH, type remote-monitoring enable. 

Automatic Certificate Renewal

Every IoT Core/Core Lite Droplet checks every 12hours once with vMIST Server (vitalpointz Managed IoT Service) for availability of new letsencrypt certificate. If new certificate is available, same is downloaded into droplet for continued working.  

Note: vMIST servie uses this Automatic Certificate Renewal API call as a heartbeat from IoT Core/Core Lite droplet. If the droplet had not made this call for 5days, the subdomain, corresponding certificate and the backed up data if any, pertaining to the droplet will be deleted permanently.
