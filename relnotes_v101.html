Whats New

New Management features

 

vMIST (vitalpointz Manged IoT) Service

Introducing a new, centralized service that provides following service to every IoT Core / Core Lite droplet spin up by customer.

- When a IoT Core/Core Lite is installed on a new droplet, Access URL is created, registered with DNS service.

- A New Letsencrypt certificate is registered and certificate downloaded into the droplet

- Periodically letsencrypt certificate is checked for expiry and if expiry is near by, certifiacate is renewed.

- Provides and maintaines space for customer droplet to store and restore the IoT Core/Core Lite data

- Upon request, provides SSH Keys (Private and Public) for each droplet. This is useful for end customer to request  remote debug/support session securely with vitalpointz support team

- Remote Monitoring service that provides vitalpointz managed service team a performance statistics of customer droplet.

Note: Subscription to vMIST Service is mandatory and free. vMIST service runs as part of vitalpointz service offering and does not require additional purchase of resource from digitalocean by customer.

 

Back up & Restore

This feature is available under CLI of the droplet. Login to CLI using SSH, type backup. This will archive the essential data from the running IoT Core/Core Lite, encrypts them, saves under vitalpointz Managert IoT Service (vMIST) Server.

Remote Debug

This feature is available under CLI of the droplet. Login to CLI using SSH, type remote-debug enable. This will enable remote SSH access of the droplet to vitalpointz support team. The SSH Key is autogenerated from vitalpointz Managet IoT Service (vMIST) server, public key stored on the droplet, private key will be made available to vitalpointz support staff. Once enabled, the SSH key will be active for 72hours. Post that, the key is automatically removed from the droplet.

If the user wants to terminate the remote ssh window, they can run remote-debug disable to stop the same.

At any time, type remote-debug status to see the status.

Remote Monitoring

This feature is available under CLI of the droplet. Login to CLI using SSH, type remote-debug enable.

Automatic Certificate Renewal

Every IoT Core/Core Lite Droplet checks every 12hours once with vMIST Server (vitalpointz Managed IoT Service) for availability of new letsencrypt certificate. If new certificate is available, same is downloaded into droplet for continued working.

Note: vMIST servie uses this Automatic Certificate Renewal API call as a heartbeat from IoT Core/Core Lite droplet. If the droplet had not made this call for 5days, the subdomain,corresponding certificate and the backedup data if any, pertaining to the droplet will be deleted permanently

 

New Device Features

Device certificate Revocation/ Re-issue

Now, device certificate can be revoked and re-issued from the UI.

 

Virtual Box FogNode Edition improvments

Bug relating to Event reception at FogOS (virtual box edition) is fixed.

 

Bug Fixes

Mongo Db error

under heavy message I/O load, the UI login could fail with the error 'Topology destroyed'. The error is fixed in this release.
