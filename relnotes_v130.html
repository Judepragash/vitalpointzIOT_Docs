Introducing Comprehensive Device Cert Authority Capabilities

We are introducing vitalpointz Root CA for exclusively signing Device certificates. our vMIST Server becomes our Level1 Intermediate CA and every one of your vitalpointz IoT Core Lite instance is actually a Level2 Intermediate CA from where device and fognode certificates are signed and distributed. 
 

 

This release also supports your own Enterprise CA (or) External Offline Root CA. In this scenario, the IoT Core Lite instance becomes Intermediary CA to the external Root CA. Device and Fognode certificates are signed and derived from the IoT Core Lite Platform as in default scenario.

 Note: All web UI facing APIs are continue to be protected using Letsencrypt Certificate.
This feature only impacts the Device certificates and Device facing services.

Setting up Security regime

Vitalpointz offers a comprehensive solution to address most digital certificate requirements in an IoT environment. By default, Vitalpointz uses a multi-layered PKI across “Operation Services & Tenant Platforms”. This empowers Vitalpointz IoT platform to maximize tenant level authorization & controls while simplifying rest of the certificate management process. In addition to that Vitalpointz IoT platform also has support for Enterprise CA that ensure the chain-of-trust remains within the domain of an enterprise.  

Which Certificate Root will be trusted by IoT devices in Vitalpointz IoT platform is established during the installation wizard and remain unchanged throughout the life of the platform. The standard Vitalpointz IoT platform installation wizard provides two ways to bring up the Platform CA (L2 Intermediate) namely VPZ_CA & BYO_CA. 

 

Starting with installation

Getting your own copy of vitalpointz IoT Core Lite on your droplet is largely same as before. See here to read again. 
 
Following section describes the changes introduced in the version v1.3.

 
typing install will start the installation wizard.

VPZ_CA 

This is a suitable for most users & doesn’t need the users to have any knowledge on PKI or Digital certificate. User just need to choose “option 1” on prompt “Would you prefer 

Vitalpointz PKI to manage Device Certificate in this platform?”. Upon selecting that, the installation will continue with no further question. A CSR is generated in the background and got it signed through a Vitalpointz L1 intermediate available at VMIST service of that Geolocation.  

 

BYO_CA 

This option is for Enterprises with own “Enterprise Root CA”. Upon choosing option 2, Platform installation wizard generates a CSR with the data given during the installation wizard. 

 

The CSR is printed on the console itself. The generated CSR is a PEM encoded Certificate Signing Request. User is required to copy the contents into a .pem file and sign with the enterprise root key. Meanwhile the installation wizard remains open or exit temporally.

 

If exited, it can be resumed from the same place later on with same “install” command. While signing, it’s important to keep in mind that the Signing Request is for an intermediate certificate which will sign device certificate going further. This should be signed for appropriate lifetime and depth.

Once the CSR is signed, the root & signed-certificate/certificate-chain must be imported to the platform. User need to copy the PEM encoded content of root certificate & paste in the console.

 

Subsequently the certificate chain (single cert or more than one) need to paste and save. The wizard will do little verification on validity of the certificate and installation will start.

 

 
The VESPa agent that is downloadable from the platform will have its own certificate store which will contain the root certs necessary to access the platform services. User just need to download the code & flash the IoT device. Just supply the auth credentials generated by platform & the device will swiftly onboard to the platform.  
Vitalpointz IoT platform doesn’t require its users to have expertise in TLS or Digital certificate. At the same time if a user wants to make it a part of their existing PKI, that’s very much doable. Certificate & Key management is the core part of IoT security. Vitalpointz IoT platform offers a range of tools for IoT developers to design & deploy their customized IoT security schemes effortlessly.

Broadcaseter Service

A new service called 'broadcaster service' is introduced in this release. This service is useful to send a message from platform to ALL the devices.

 

Generally braodcasting messages to devices involve following steps

1) write a function in NodeRED or Workloads that generate messages to be broadcaseted

2)Wire the message to Broadcast Service.

3) The broadcaster service tries to parse the message received into JSON object. If Parsing fails, the message is sent to ALL devices and ALL Fog Nodes.

3.1.) if the JSON is parsed successfully and it looks for a key called 'broadcast'. If the key is set to'device' the service sends the message only to the devices connected to the platform.

3.2.) If the key is set to'fognode' the service sends the message only to the FogNodes connected to the platform.
