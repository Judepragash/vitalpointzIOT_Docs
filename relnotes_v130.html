Introducing Comprehensive Device Cert Authority Capabilities

We are introducing vitalpointz Root CA for exclusively signing Device certificates. our vMIST Server becomes our Level1 Intermediate CA and every one of your vitalpointz IoT Core Lite instance is actually a Level2 Intermediate CA from where device and fognode certificates are signed and distributed. 
 

 

This release also supports your own Enterprise CA (or) External Offline Root CA. In this scenario, the IoT Core Lite instance becomes Intermediary CA to the external Root CA. Device and Fognode certificates are signed and derived from the IoT Core Lite Platform as in default scenario.

 Note: All web UI facing APIs are continue to be protected using Letsencrypt Certificate.
This feature only impacts the Device certificates and Device facing services.

Broadcaseter Service

A new service called 'broadcaster service' is introduced in this release. This service is useful to send a message from platform to ALL the devices.

 

Generally braodcasting messages to devices involve following steps

1) write a function in NodeRED or Workloads that generate messages to be broadcaseted

2)Wire the message to Broadcast Service.

3) The broadcaster service tries to parse the message received into JSON object. If Parsing fails, the message is sent to ALL devices and ALL Fog Nodes.

3.1.) if the JSON is parsed successfully and it looks for a key called 'broadcast'. If the key is set to'device' the service sends the message only to the devices connected to the platform.

3.2.) If the key is set to'fognode' the service sends the message only to the FogNodes connected to the platform. 

See here for detailed documentation.
