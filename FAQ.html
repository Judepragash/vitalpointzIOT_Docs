Vitalpointz IoT Core, Core-lite FAQs

 

Q. What is Vitalpointz IoT Core/Core-lite

Vitalpointz IoT Core/Core-lite is a managed cloud platform to administrate connected devices, IoT infrastructure and services.

 

Q. What are the features of Vitalpointz IoT Core

Vitalpointz IoT Core hosts an array of services for complete lifecycle management of IoT Devices and IoT Infrastructure. Essential Feature listed below

 

Device Onboarding: Vitalpointz IoT Core platform facilitates easy onboarding of IoT devices to platform for microcontroller/microprocessor both agent and agentless way. Onboarding is supported through secure as well as unsecure channels. 

 

Support for IoT Messaging Protocols: Vitalpointz IoT Core platform supports standard IoT protocols like MQTT, HTTP & WebSocket, both secure and unsecure variants.

 

Message Processing & Routing: Vitalpointz IoT core provides a bunch tools to process, consume the messages within the platform & in-platform apps or forward to an external one or both.

 

Security & OTA: Vitalpointz IoT core can add a layer of security to devices with a secure communication channel even for plaintext-communication only devices and firewall rules at both edge and cloud.

 

Q. How Vitalpointz IoT Core-lite is different from IoT Core

IoT Core-lite is the best place to start to evaluate Vitalpointz offering. IoT Core-lite can work on a cloud instance of 2 core CPU & 4 GB RAM which is effectively half to what IoT Core needs. IoT Core-lite don’t have the OTA as a major feature.

Q. How do I get started using Vitalpointz IoT Core

Sign up for an account with Digitalocean if you don’t have one already. Use the referral link to get a trial credit of $100 for 2 months.

 

Spinning a IoT core/core-lite in Digitalocean Cloud: Go to the Marketplace of Digitalocean, find Vitalpointz IoT Core/Core-lite and proceed with creation of a droplet with specifications given below

Droplet: Standard

Min Spec: IoT Core-lite – 2 Core CPU, 4GB RAM, IoT Core – 4 Core CPU, 8GB RAM

Optionally: Select the Private networking and monitoring

Rest can be selected based on preferences of user

First time login and initial config wizard: Once the droplet is available, login to it through SSH. For the first time login the SSH will run into an initial config wizard with questions appearing sequentially. In the end of the wizard, you’ll get the URL to access the platform and login credential mailed to your mailbox.

 

Onboarding IoT device: To use Vitalpointz IoT agent (VESPa), visit to the Github project of Vitalpointz IoT and download what is relevant to your device. Get one of the supported device for the agent or just get the node-red sketch to simulate few in your computer. Create equal no of devices  you want to onboard or simulate in the platform. Take a note of login key or auth token generated and burn them into the devices/simulation program. Start the devices/simulation and voila!. You’ll see the message received at the platform.

 

Visualising messages, routing & processing: Each device onboarded to the IoT Platform is assigned with a dedicated topic. Messages published to that specific topic and subsequent subtopics can be seen by selecting the topics in the message logger. Messages from multiple devices can be aggregated to node-red, in-cloud apps or other external message subscriber.

 

Q. Why message forwarding & routing needed? How is it done?

Most sensor and devices data need to be aggregated & forwarded to applications and other subscribers. Routing takes care of the simplified forwarding management in the cloud. The routing rules are manged by rules engine in the cloud

 

Q. What are the things can be done with rules engine?

In addition to forwarding rule management, rules engine handles events & actions. Rules engine can be configured to take an action like redirection, notification & forwarding.

 

Q. What are the protocols and port numbers used by IoT Core?

The control communication across Vitalpointz IoT Core & devices are through http and https protocols based on what device supports. Data communication port is different for every user and can be seen at the platform UI.

Q. What is Node-Red?

Node-red is a flow-based visual programming tool built on Node.Js. In vitalpointz IoT Platform, the hosted Node-red can be leveraged to read message, plot charts, run business logic or forward to other services. 

 

Q. What is out-port?

Out Port is a service under vitalpointz IOT Core/Core Lite using which you can export your device data or locally/programmatically generated data to external 3-rd party service.

 

Q. What protocol / interface Out-Port service uses?

Currently, Out Port Service uses REST API. It supports PUT, DELETE, GET and POST methods.

Q. How to scale for more devices?

When you install a fresh IOT Core/Core Lite instance, 1024 device licenses are bundled with it. That means you can add up to 1024 devices without requiring additional license. If you want to onboard more devices, In IOT Core/Core Lite web portal, click on top right-hand corner burger menu, click ‘License Manager’ and click ‘Request License’ button. This will send a license request message to vitalpointz managed services team. We will generate a license and send you by email.

Q. How do you ensure the confidentiality & integrity of the data sent by the sensors?

Vitalpointz IOT Core/Core Lite offers MQTT over TLS and HTTPS as standard and basic interfaces for your device to publish/subscribe messages to the platform. Certificates are issued for each device protecting individual device.

 

Q.My devices are not TLS capable. Is there a way to publish/susbcribe using plain text interfaces?

Yes, we support MQTT plaintext and HTTP interfaces. However, you need to enable this explicitly. User should be added, enable the unsecure access privilege for that user to allow unsecure access. We highly recommend you to use TLS wherever possible.

 

Q. Do you store my data? Do you have access to my data?

No. We do not store your device data.

Your device data reaches your instance of IOT Core/IOT Core Lite. You can choose to consume it on the droplet using hosted NodeRED tool or forward to external service using Out Port service.

Q. What is VESPA?

VESPA stands for Vitalpointz Edge Service Platform Agent. They are device agents that run on IOT Devices, Gateways etc.

 

Q. What are the variants of VESPA available?

Agent software comes in following variants:

VESPA-L : Linux variant, Runs as a user process in Linux

VESPA-G : Gateway variant available in 2 editions. As an appliance and as virtual box image

VESPA-U: Microcontroller variant, available as arduino sketches and Node-RED flows that simulate IOT devices.

VESPA-A: Android variant

Q. Is there a way to simulate one or many devices?

Yes. It is possible to simulate one or many devices. In fact, this is the best way to learn device authentication procedure and how vitalpointz IOT Core treats devices.

Our simulators are open sourced under MIT license. They are runnable under NodeRED.

You need to install a NodeRED in you VM or laptop, import the flows and follow the steps provided.

 

Q. How do I download the VESPA software / simulators?

Log into your droplet that runs vitalpointz IOT Core/Core Lite. Add Device under Device Tab.

Click ‘download image’ menu under Device panel. This will open a separate page or start to download the image depending on the variant.

 

Q. Is my IOT Core/Core Lite web portal secured? Do I need to provide my own certificate?

Your instance of IOT Core/IOT Core Lite Web UI is secured by Letsencrypt certificate. You do not need to purchase or bring your own certificate.

 

Q. Do I need to configure DNS or Letsencrypt myself?

No. When you spin up a droplet with vitalpointz IOT Core/Core Lite, automatically a subdomain is created, and DNS configured appropriately. Let’sencrypt certificates are generated and provisioned on your instance. So, you do not need to do anything.

 

Q. Do I need to renew Let’sencrypt certificates manually?

No. Certificate renewal is automatically done. So, you do not need to do anything.
